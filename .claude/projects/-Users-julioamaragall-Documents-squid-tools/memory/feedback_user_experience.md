The audience is life sciences researchers who don't know programming. The GUI must require zero training (QuPath requires training, that's bad).

**Why:** PIs don't have time to train their lab members. If the software needs a manual, it has failed.

**How to apply:**
- Every button needs a tooltip
- Processing operations should have sensible defaults populated from metadata
- No jargon in the UI (say "Remove background" not "sep.Background estimation")
- The download page (like cephla-downloads.pages.dev/ndviewer-light/) should let users pick what they need without understanding packages
- The checkbox installer translates to: user clicks what they want on a web page, downloads a bundle
- Future: sophisticated web page for seamless package selection and download
